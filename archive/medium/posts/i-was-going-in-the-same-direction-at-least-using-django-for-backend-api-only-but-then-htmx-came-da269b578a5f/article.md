# I was going in the same direction, at least using Django for backend/API only, but then htmx came…

I was going in the same direction, at least using Django for backend/API only, but then htmx came along, and it solved all my needs for a great UI experience and allowed me to continue to use all the batteries included in django, like form generation/validation etc. Web development using javascript is currently at a difficult stage with lots of problems related to npm package management, speed decline in each new release of node etc. The creator of node and next have both moved to Go as programming language. My experience using Vue and node professionally for a big project was a very mixed experience, but luckily I can still use Django for all my personal projects, and that won't change anytime soon due to the arrival of htmx :-)
