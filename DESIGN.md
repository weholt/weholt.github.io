# Weholt.org Design System

Derived from Open Design `editorial` system via `design-brief` skill.

## Visual Theme & Atmosphere

- Mood: editorial, professional minimal
- Feel: calm, confident, readable — a senior developer portfolio, not a SaaS landing page
- References: magazine layouts, restrained product pages, technical writing

## Color Palette & Roles

- Background: `#0a0f14`
- Surface: `#121a22`
- Surface elevated: `#1a2430`
- Text primary: `#eef2f6`
- Text secondary: `#94a3b8`
- Accent: `#5b9fd4`
- Accent hover: `#7ab8e8`
- Border: `rgba(238, 242, 246, 0.1)`

Light theme (`clean-light`):

- Background: `#f7f5f1`
- Surface: `#ffffff`
- Text primary: `#1a1a1a`
- Text secondary: `#5c5c5c`
- Accent: `#1f6feb`

## Typography Rules

- Display: Fraunces, 600, clamp(2rem, 4.5vw, 3.25rem)
- Body: "DM Sans", 400, 1rem/1.65
- Mono: "IBM Plex Mono", 400, 0.875rem
- Headings use display; UI labels and body copy use body

## Layout Principles

- Max width: 1080px
- Section spacing: 72px
- Content padding: 24px / 40px
- 8pt spacing rhythm
- One primary layout visible at a time

## Do's and Don'ts

- DO use whitespace before borders and shadows
- DO keep accent color to links, kickers, and one CTA per viewport
- DO ensure headlines wrap — never clip overflow text
- DON'T use purple-blue gradient hero backgrounds
- DON'T use Inter as the display face
- DON'T randomize layout/theme on first visit
